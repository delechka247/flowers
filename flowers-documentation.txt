1. Введение: 
	 Проект, разрабатываемый dev1 - Кукушкиной Аделей и dev2 - Назарычевым Владиславом, под руководством PO - Сергея Башаркина, цель которого - предоставление информации о  интернет-магазине, где каждый пользователь, быстро и не выходя из дома может заказать товары (цветы, горшки, букеты) на дом или на само-вывоз. Заказ осуществляется по id товара и добавлению его в корзине, далее для оформления заказа нажмите "оформить заказ" и укажите нужные данные для его оформления. 

2. Нефункциональные требования: 
	2.1 Требования к численности и квалификации персонала (пользователей) 
Для администрирования сайта необходимы базовые знания по настройке Web-сервера и сервера базы данных. При этом администратор сайта не должен быть специалистом в настройке и администрировании этих серверов, т.е. Система должна быть разработана таким образом, чтобы процедуры развертывания сайта, установки обновлений, резервного копирования были реализованы с максимальным удобством и не требовали специальных знаний от технического специалиста. 

Интерфейс сайта должен быть построен таким образом, чтобы пользователю знакомому с операционными системами семейства Windows было просто разобраться с элементами интерфейса. Общая концепция интерфейса должна быть организована таким образом, чтобы обеспечить комфортную работу пользователей и интуитивно понятное взаимодействие с Системой. 

	2.2. Требования к эргономике и технической эстетике 
Взаимодействие пользователей с прикладным программным обеспечением, входящим в состав сайта, должно осуществляться посредством визуального графического интерфейса. 

Интерфейс сайта должен быть понятным и удобным, не должен быть перегружен графическими элементами и должен обеспечивать быстрое отображение экранных форм. Навигационные элементы должны быть выполнены в удобной для пользователя форме. Ввод-вывод данных в Системе, прием управляющих команд и отображение результатов их исполнения должны выполняться в интерактивном режиме. Интерфейс сайта должен соответствовать современным эргономическим требованиям и обеспечивать удобный доступ к основным функциям и операциям сайта. 

Интерфейс должен быть рассчитан на преимущественное использование манипулятора типа «мышь», то есть управление Системой преимущественно должно осуществляться с помощью набора экранных меню, кнопок, значков и т. п. элементов. Клавиатурный режим ввода должен использоваться главным образом при заполнении и/или редактировании полей экранных форм. 

Все надписи экранных форм, а также сообщения, выдаваемые пользователю, должны быть на русском языке. 
функционал должен обеспечивать корректную обработку ситуаций, вызванных неверными действиями пользователей, неверным форматом или недопустимыми значениями входных данных. В указанных случаях сайт должна выдавать пользователю соответствующие сообщения, после чего возвращаться в рабочее состояние, предшествовавшее неверной (недопустимой) команде или некорректному вводу данных. 
функионал должен обеспечивать контроль ввода данных пользователем на предмет наполненности обязательных полей и корректности формата введенных данных. 

Экранные формы должны проектироваться с учетом требований по унификации:  
– Все экранные формы пользовательского интерфейса должны быть выполнены в едином графическом дизайне, с одинаковым расположением основных элементов управления и навигации. 
– Для обозначения сходных операций должны использоваться сходные графические значки, кнопки и другие управляющие (навигационные) элементы. Термины, используемые для обозначения типовых операций (добавление информационной сущности, редактирование поля данных), а также последовательности действий пользователя при их выполнении, должны быть унифицированы. 
– Внешнее поведение сходных элементов интерфейса (реакция на наведение указателя «мыши», переключение фокуса, нажатие кнопки) должны реализовываться одинаково для однотипных элементов. 

	2.3 Защита информации от несанкционированного доступа 
Сайт относится к группе многопользовательских ИС с разными правами доступа. Размещение в Системе конфиденциальных данных не предусматривается. При разработке интерфейсов взаимодействия со смежными системами должны быть предприняты меры по исключению загрузки в Систему персональных данных или иной охраняемой законом информации ограниченного доступа. 
Идентификация пользователей должна осуществляться по связке «имя пользователя и пароль». 

	2.4 Требования к организационному обеспечению: 

К работе с сайтом должны допускаться сотрудники, имеющие навыки работы на персональном компьютере, ознакомленные с правилами эксплуатации и прошедшие обучение работе с Системой. 

	
3. Функциональные требования: 

	3.1. Навигация по сайту и описание страниц: 

1. Главная страница 

добро пожаловать в наш магазин, информация о стоимости доставки и т.д. 

2. Каталог товаров - цветы, букеты, горшки или показать все товары 

3.Страница со списком товаров - фильтр и далее все товары по списку 

4. Страница определенного товара - Название, информация о нем (цена, кол-во и т.д) 

5. Страница регистрации/входа - вводятся имя пользователя(обяз.), пароль(обяз.), доп. данные 

6. Личный кабинет - информация о текущем пользователе, история заявок 

7. Страница, доступная только админам, чтобы новые товары добавлять на сайт и в базу данных товары, и изменять кол-во товаров в наличии 

8. Оставить отзыв к товару - страница заполнения отзывов 

9. комната админов -здесь можно посмотреть все заявки, сборы заказов и доставку 

10. Оформление заказа - пошаговое создание заявки 

	3.2 Система должна выполнять(иметь) следующие функции: 

	1) Сайт должна предоставлять возможность: 

		а) удаления/добавления товаров 

		б) изменения количества товаров 

		в) загрузка фотографий к товару (при создании) 

	2) Функция поиска данных пользователей, фильтрация данных 

	3)Ведение баз данных (приложение в пункте 4.1) 

	Программное обеспечение баз данных - СУБД MySQL 

	4) Сохранение данных в cookies 

	5) Использование сессий 

	3.3 Особые требования к оформлению: 

	Каждая страница сайта в верхней части должна содержать шапку - 

изображение с названием сайта и определенным стилем, панелью из 5 пунктов 

("Главная", "О нас", "мои заявки", "Каталог", Личный кабинет"), 

	При нажатии личный кабинет, если пользователь не авторизирован, выплывает страница для входа/регистрации, иначе информация о текущем пользователе, текущие заявки. 

	Кнопки с шапки (панели) остаются на всех страницах - отправляют на страницы соответствующим своим названиям.  

	На странице "Личный кабинет" имеется кнопка редактировать профиль, где пользователь может изменить свои данные. 

	Кнопки цветы/горшки/показать все находятся на странице "Каталог" - показывают товары соответственно по определенному типу - только цветы/только горшки/все доступные товары на странице. 

	Панель фильтрации находится на странице "Список товаров" - фильтр по определенным свойствам (по типу, цене и т.д.). 

	Иконка каждого товара на странице "список товара" - перенесет нас на персональную страницу выбранного товара. 

	На персональной странице товара находится его описание, и кнопка "оформить заказ" - далее пользователь выбирает количество и данный товар добавляется в "заявки" и добавляется в заказ. 

	В самом низу "страницы товара" находится кнопка оставить отзыв (сама функция только для авторизированных пользователей) - которая переносит нас на страницу с заполнением отзыва о данном товаре. 

	С помощью добавления “/admin”  к пути нашего сайта (например flowers.com/admin) можем получить доступ к "Комнате админов", но только для пользователя, авторизированного как админ -  где получаем доступ к списку всех заявок, сборов заказов и доставок. 

	в той же "Комнате администраторов" есть кнопка "добавить товар", которая переносит нас на страницу добавления товара, где появляются поля для заполнения названия, кол-ва, кнопка "прикрепить фото", и внизу окончательная кнопка "добавить" - для отправки товара в базу данных и в каталог. 

4. Приложения: 

	4.1 База данных: 

11. Юзер 

  колонки: id(primary key), email, пароль, имя, номер телефона, день рождения, возможность изменять сайт. 

2. Товар 

  колонки: id(primary key), тип(цветок, букет, горшок и тд.), цена, кол-во в наличии, рейтинг товара. 

3. Цветок 

  колонки: id(foreign key из товара), название, цвет, длина стебля. 

4. Букет 

  колонки: id(foreign key из товара), описание(из каких цветов состоит), кол-во цветов внутри, дата сборки. 

5. Горшок 

  колонки: id(foreign key из товара), цвет, диаметр, материал 

6. Отзывы о товаре 

  колонки: id(primary key, номер отзыва), id(foreign key из юзера), id(foreign key из товара), оценка(от 1 до 5), текст, дата отправки. 

7. Заявка на товар 

  колонки: id(primary key, номер завяки), id(foreign key из юзера), id(foreign key из товара), кол-во товара, стоимость, дата создания заявки, тип(самовывоз или доставка), адрес(если самовывоз то адрес магазина), время доставки/получения.
